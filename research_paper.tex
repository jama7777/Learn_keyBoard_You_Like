\begin{abstract}
Proficiency in touch typing remains a critical competency in the
digital era, yet traditional typing tutors rely on repetitive, static
content that fails to sustain user engagement over extended sessions.
This paper introduces \textbf{TypeMaster AI}, an advanced multimodal
typing tutor that harnesses Large Language Models (LLMs) to deliver
personalized and interactive learning. By integrating the Google
Gemini 2.0 API with an OpenRouter failover gateway, the system
provides: (i) on-demand lesson generation from any user-defined
topic; (ii) multimodal content extraction from uploaded images and
PDFs; (iii) real-time grammar correction for custom text; and (iv)
an immersive \emph{Melody Typing} mode that synthesizes musical
notes via the browser's Web Audio API. High-frequency keyboard events
are handled through React 19's concurrent rendering pipeline,
ensuring sub-millisecond UI responsiveness at typing speeds exceeding
100 WPM. Informal user trials indicate a measurable improvement in
sustained practice duration compared to conventional fixed-text
tutors. The live application is accessible at:
\url{https://learn-key-board-you-like.vercel.app/}.
\end{abstract}

\begin{IEEEkeywords}
Typing tutor, Multimodal AI, Large Language Models, Gemini API,
OpenRouter, Web Audio API, React.js, Gamified learning,
Human-Computer Interaction.
\end{IEEEkeywords}

\section{Introduction}

Touch typing has evolved from a clerical skill into a core digital
literacy requirement. A proficient typist can sustain 60--80 WPM
with high accuracy, dramatically improving productivity across
engineering, academia, and creative domains~\cite{brown2012}.
However, the majority of freely available web-based typing tutors
(e.g., Typing.com, 10FastFingers) present users with a fixed corpus
of generic sentences, leading to disengagement and performance
plateaus after initial gains.

The advent of Generative AI presents an opportunity to break this
ceiling. A model that can produce contextually relevant, topically
varied, and appropriately formatted practice text on demand can keep
learners engaged indefinitely. Further, the integration of auditory
feedback---a technique well-established in cognitive psychology as
enhancing motor skill retention~\cite{mayer2014}---provides an
additional motivational channel.

TypeMaster AI addresses these gaps through a three-pronged approach:
\begin{enumerate}
  \item An \textbf{AI Content Engine} that generates lessonspecific
        content from any topic the user supplies, including code,
        prose, and drill sequences.
  \item A \textbf{Dual-API Orchestration Layer} that guarantees
        service continuity via automatic failover from Gemini to OpenRouter.
  \item A \textbf{Web Audio Synthesis Engine} that transforms each
        keystroke into a musical note, creating an immersive and
        game-like typing environment.
\end{enumerate}

\section{Related Work}

Several prior systems have explored AI-assisted language learning.
Duolingo employs spaced-repetition algorithms to optimize vocabulary
retention~\cite{mayer2014}. Keybr.com generates adaptive lessons
based on character frequency analysis of natural language corpora.
However, neither system applies generative LLMs for real-time content
creation, nor do they incorporate multimodal (audio-visual) feedback.

In the domain of motor-skill learning, Ericsson's deliberate practice
framework~\cite{brown2012} emphasizes the necessity of focused,
feedback-rich repetitions. TypeMaster AI operationalizes this
framework digitally: the AI generates the topic, the Virtual Keyboard
provides visual cues, and the Melody Engine provides instant auditory
reinforcement per keystroke.

\subsection{Concurrent UI and State Management}

All typing state---current character index, error count, WPM, and
accuracy---is managed via React's \texttt{useState} and \texttt{useRef}
hooks. To prevent event-handler drift at high WPM, the keyboard
listener is memoised with \texttt{useCallback}, ensuring a stable
reference across render cycles. A \texttt{useEffect} hook attaches and
detaches the global \texttt{keydown} listener idempotently, preventing
memory leaks during session transitions.

User progress and AI-generated lesson history are persisted in the
browser's \texttt{localStorage} through a dedicated
\texttt{LocalHistoryService} module, allowing returning users to
resume where they left off without requiring authentication.

\subsection{Virtual Keyboard and Finger Guidance}

Each key on the on-screen QWERTY layout is pre-assigned to one of ten
logical finger zones (L\_PINKY through R\_PINKY) in a static
\texttt{CHAR\_TO\_KEY\_MAP}. On each keystroke, the component
highlights the next target key and the corresponding finger position
in the associated color, providing ergonomic guidance during practice.
Shift-state detection enables accurate mapping of symbols and
uppercase letters to the correct hand.

\subsection{Real-time Auditory Synthesis}

The Audio Service instantiates a single \texttt{AudioContext} on
first user interaction (satisfying browser autoplay policies). For
each correct keystroke, it:
\begin{enumerate}
  \item Creates an \texttt{OscillatorNode} with \texttt{type = "sine"}.
  \item Creates a \texttt{GainNode} to shape the amplitude envelope
        (sharp attack, exponential decay over 300 ms).
  \item Sets the oscillator frequency to the precomputed value for
        the current note in the melody sequence (e.g., C4~=~261.63 Hz).
  \item Connects the graph: \texttt{Oscillator} $\rightarrow$
        \texttt{Gain} $\rightarrow$ \texttt{AudioContext.destination}.
  \item Schedules the oscillator to stop after the decay completes.
\end{enumerate}
This approach avoids the overhead of pre-recorded samples and produces
near-instantaneous audio with no perceptible latency, which is
critical for maintaining the user's rhythmic flow.

\section{AI Content Engine and LLM Orchestration}

\subsection{Lesson Format Generation}

The AI Manager sends a structured prompt to the LLM that encodes
both the \emph{topic} and the desired \emph{output format}. The
system supports six lesson formats:

\begin{table}[h]
\centering
\caption{Supported AI Lesson Formats}
\begin{tabular}{ll}
\toprule
\textbf{Format}       & \textbf{Description} \\
\midrule
Paragraph             & General prose, $\sim$50 words \\
Story                 & Short narrative, first-person \\
Business Letter       & Formal letter body structure \\
Abstract              & Dense academic writing style \\
Code (Python)         & Valid Python with comments \\
Code (JavaScript)     & Valid JS with inline comments \\
\bottomrule
\end{tabular}
\end{table}

The prompt also includes a special-case branch: if the topic resembles
a key-drill input (e.g., ``a, s, d, f''), the model produces a
repetitive drill sequence rather than prose, maximising the density of
target characters in the output.

\subsection{Prompt Engineering and Output Sanitization}

Raw LLM outputs frequently contain Markdown decorators (e.g.,
\texttt{```python}) and conversational filler. The service
post-processes every response with a regex pipeline
(\texttt{/```[a-z]*\\n?/g}) to strip code fences and return clean,
typable text. This step is critical for code-format lessons where
indentation must be preserved but Markdown wrapping must be removed.

\subsection{Grammar Correction Pipeline}

Custom text pasted by the user is silently routed through a grammar
correction prompt before being loaded into the typing area. The
prompt instructs the model to return \emph{only} the corrected text,
preserving original paragraph structure and tone. This ensures
practice material is free of distracting errors that could
inadvertently teach incorrect typing patterns.

\subsection{OpenRouter Failover and Rate-Limit Protection}

The \texttt{getApiKey()} function detects the type of API key at
runtime by inspecting its prefix: keys beginning with
\texttt{sk-or-} are routed through the OpenRouter REST endpoint
(\texttt{openrouter.ai/api/v1/chat/completions}), while standard
Gemini keys use the official \texttt{@google/genai} SDK.

This dual-path strategy provides two critical reliability guarantees:
\begin{itemize}
  \item \textbf{Rate-limit bypass:} Gemini's free tier enforces
        strict Requests-Per-Minute (RPM) and Tokens-Per-Minute (TPM)
        quotas. When throughput is exhausted, the orchestrator
        automatically re-routes the request to OpenRouter's shared
        model pool without interrupting the user session.
  \item \textbf{Model versioning resilience:}  OpenRouter abstracts
        model identifiers, allowing the system to fall back to an
        alternate compatible model (e.g.,
        \texttt{google/gemini-2.0-flash-exp:free}) if the primary
        model ID becomes deprecated or unavailable.
\end{itemize}

\subsection{Multimodal Input Processing}

Gemini's native multimodal support is leveraged to extract typed
practice content from user-uploaded files. Binary file data (image or
PDF) is encoded as Base64 and embedded in the
\texttt{inlineData} field of the API request. The model returns a
plain-text transcription that is immediately loaded into the typing
area, allowing users to practice content from textbooks, slide decks,
or handwritten notes.

\section{Gamified Learning: Melody Typing Mode}

\subsection{Musical Library and Note Mapping}

The system ships with a curated library of over 50 predefined
melodies spanning classical compositions, pop culture themes, and
educational nursery rhymes. Each song is represented as an ordered
array of note-frequency mappings (e.g., C4~=~261.63 Hz, A4~=~440

\subsection{Reinforcement Feedback Loop}

Correct keystrokes advance the melody sequence and play the
corresponding synthesized note. Incorrect keystrokes do not advance
the melody and produce a brief low-frequency error tone, providing
immediate negative reinforcement without interrupting the practice
flow. This design is consistent with operant-conditioning principles
where feedback immediacy maximises learning signal~\cite{mayer2014}.

Users can also select lessons by specific keyboard zone (Home Row,
Top Row, Bottom Row, Numbers, Symbols) and each zone is paired with a
recommended melody whose rhythm complements the key-reach patterns of
that zone.

\section{Deployment and Performance}

The application is deployed on \textbf{Vercel}'s edge network using
a CI/CD pipeline connected to the GitHub repository. The Vite build
system tree-shakes unused TypeScript modules and produces a
production bundle under 250 KB (gzipped). Environment variables
(including the API key) are managed through Vercel's encrypted
secrets store, ensuring no credentials are exposed in the
client-side bundle.

Key performance characteristics observed during testing:
\begin{itemize}
  \item \textbf{Time-to-First-Lesson:} $<$1.5\,s on a standard
        broadband connection.
  \item \textbf{Keystroke-to-Audio Latency:} $<$5\,ms (bounded by
        the Web Audio API scheduler).
  \item \textbf{LLM Response Time:} 1--4\,s for Gemini direct;
        2--6\,s via OpenRouter (varies by model load).
\end{itemize}

\section{Conclusion and Future Work}

TypeMaster AI demonstrates that the convergence of Generative AI,
modern web frameworks, and real-time audio synthesis can
fundamentally transform a decades-old educational application domain.
By solving concrete engineering challenges---API rate-limit
resilience, sub-millisecond audio latency, and concurrent UI
state---the system delivers a robust, production-ready platform for
personalized typing skill acquisition.

Future work will explore: (i) \textbf{Adaptive difficulty} using
per-user WPM and error-rate analytics to dynamically adjust lesson
complexity; (ii) \textbf{Live speech-to-text drills} leveraging the
Web Speech API to generate typing content from spoken prompts;
(iii) \textbf{Multiplayer racing mode} to introduce competitive
incentives; and (iv) \textbf{Custom melody creation} allowing users
to define their own note sequences.

The production deployment is live at:
\url{https://learn-key-board-you-like.vercel.app/}.
